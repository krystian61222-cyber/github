# Analiza inwestycji Andreessen Horowitz (a16z) w GitHub Inc.

**Data opracowania:** czerwiec 2025

---

## Wprowadzenie

Andreessen Horowitz (znany rowniez jako a16z) to amerykanski fundusz venture capital zalozony w 2009 roku przez Marca Andreessena i Bena Horowitza. Fundusz specjalizuje sie w inwestycjach technologicznych na wczesnym i pozniejszym etapie rozwoju spolek, obejmujac sektory takie jak oprogramowanie, infrastruktura internetowa, kryptowaluty, biotechnologia oraz media. Od momentu zalozenia a16z zgromadzil aktywa o wartosci przekraczajacej kilkadziesiat miliardow dolarow i jest uznawany za jeden z najbardziej wplywowych funduszy venture capital na swiecie.

GitHub Inc., zalozony w 2008 roku przez Toma Preston-Wernera, Chrisa Wanstratha i PJ Hyetta, szybko stal sie dominujaca platforma do hostingu repozytoriow kodu zrodlowego opartych na systemie kontroli wersji Git. W momencie inwestycji a16z w 2012 roku GitHub posiadal juz miliony uzytkownikow i byl postrzegany jako kluczowy element infrastruktury wspolczesnego procesu tworzenia oprogramowania. Platforma oferowala model freemium -- darmowy hosting repozytoriow publicznych oraz platne konta dla repozytoriow prywatnych i zespolow komercyjnych.

Inwestycja a16z w GitHub stanowi jeden z najbardziej emblematycznych przykladow udanej inwestycji venture capital w sektorze narzedzi deweloperskich. Ponizszy dokument przedstawia szczegolowa analize tej transakcji, jej kontekstu, przebiegu oraz wynikow.

---

## 1. Wartosc inwestycji

W lipcu 2012 roku Andreessen Horowitz dokonal inwestycji w wysokosci **100 milionow dolarow amerykanskich** w GitHub Inc. Byla to runda finansowania Series A, w calosci poprowadzona przez a16z. Calkowity rozmiar rundy wynosil 100 milionow dolarow.

Kluczowe parametry transakcji:

- **Kwota inwestycji:** 100 000 000 USD
- **Typ rundy:** Series A
- **Lead investor:** Andreessen Horowitz
- **Wycena pre-money:** okolo 750 milionow USD
- **Data zamkniecia:** lipiec 2012

Warto podkreslic, ze byla to jedna z najwiekszych rund Series A w historii branzy technologicznej w tamtym okresie. GitHub, przed ta inwestycja, dzialal przez cztery lata bez zewnetrznego finansowania venture capital, co stanowilo rzadkosc wsrod szybko rosnacych spolek technologicznych. Firma byla rentowna i generowala przychody z subskrypcji platnych kont, co pozwolilo jej osiagnac znaczaca skale bez rozwodnienia udzialow zalozycieli.

Decyzja o przyjeniu zewnetrznego kapitalu byla motywowana nie tyle potrzeba finansowania biezacej dzialalnosci, co checia przyspieszenia ekspansji na rynek korporacyjny oraz skorzystania z operacyjnego wsparcia, jakie oferowalo a16z w ramach swojego modelu "high-touch".

## 2. Branza i cel inwestycji

### Sektor

GitHub operowal na przecieciu kilku sektorow technologicznych:

- **Narzedzia deweloperskie (Developer Tools)** -- platformy i uslugi wspierajace proces tworzenia oprogramowania
- **Infrastruktura rozwoju oprogramowania (Software Development Infrastructure)** -- systemy kontroli wersji, CI/CD, zarzadzanie kodem
- **Oprogramowanie jako usluga (SaaS)** -- model subskrypcyjny dostarczania oprogramowania przez internet

### Cel przeznaczenia srodkow

Pozyskane 100 milionow dolarow mialo zostac przeznaczone na nastepujace cele strategiczne:

- **Skalowanie platformy** -- rozbudowa infrastruktury technicznej w celu obslugi rosnacego ruchu i liczby repozytoriow. W momencie inwestycji GitHub hostowal juz miliony repozytoriow, a dynamika wzrostu wymagala znaczacych nakladow na infrastrukture serwerowa i sieciowa.

- **Ekspansja produktu korporacyjnego (GitHub Enterprise)** -- rozwoj dedykowanego produktu dla duzych organizacji, oferujacego hosting repozytoriow na wlasnych serwerach klienta (on-premises) z zaawansowanymi funkcjami bezpieczenstwa, audytu i zarzadzania uzytkownikami.

- **Rekrutacja talentow inzynierskich** -- pozyskanie najlepszych inzynierow oprogramowania, projektantow i menedzerow produktu w celu przyspieszenia rozwoju platformy i nowych funkcjonalnosci.

- **Ekspansja miedzynarodowa** -- rozszerzenie obecnosci GitHub na rynkach poza Stanami Zjednoczonymi, w tym budowa zespolow wsparcia i sprzedazy w kluczowych regionach (Europa, Azja).

## 3. Wyniki inwestycji

### Przejecie przez Microsoft

W czerwcu 2018 roku Microsoft Corporation oglosil przejecie GitHub Inc. za kwote **7,5 miliarda dolarow amerykanskich** w formie akcji Microsoft. Transakcja zostala zamknieta w pazdzierniku 2018 roku po uzyskaniu niezbednych zgod regulacyjnych.

### Zwrot z inwestycji dla a16z

Inwestycja a16z o wartosci 100 milionow dolarow przy wycenie okolo 750 milionow dolarow oznaczala, ze fundusz objal udzial na poziomie okolo 13,3% w spolce. Przy cenie przejecia 7,5 miliarda dolarow, szacunkowy zwrot z inwestycji wyniosl:

- **Mnoznik zwrotu:** okolo **10x** (dziesieciokrotnosc zainwestowanego kapitalu)
- **Szacunkowa wartosc udzialu przy wyjsciu:** okolo **1 miliarda dolarow**
- **Okres inwestycji:** 6 lat (2012-2018)

### Znaczenie strategiczne

Inwestycja w GitHub byla jednym z najbardziej udanych przedsiewziec inwestycyjnych a16z we wczesnym okresie dzialalnosci funduszu. Stanowila potwierdzenie tezy inwestycyjnej Marca Andreessena wyrozonej w jego slynnym eseju "Why Software Is Eating the World" (2011), zgodnie z ktora firmy dostarczajace infrastrukture dla tworcow oprogramowania beda jednymi z najcenniejszych przedsiebiorstw przyszlosci.

## 4. Osoba odpowiadajaca za transakcje

### Peter Levine -- General Partner, Andreessen Horowitz

Inwestycja w GitHub byla prowadzona przez **Petera Levine**, General Partnera w funduszu Andreessen Horowitz. Levine, posiadajacy bogate doswiadczenie w sektorze oprogramowania open source i infrastruktury IT (wczesniej CEO XenSource, VP w Citrix Systems), doskonale rozumial potencjal GitHub jako platformy laczacej spolecznosc open source z rynkiem korporacyjnym.

Kluczowe fakty dotyczace roli Petera Levine:

- Pelnii funkcje lead partnera odpowiedzialnego za transakcje ze strony a16z
- Dolaczyl do **rady dyrektorow (Board of Directors) GitHub** po zamknieciu rundy
- Aktywnie wspieraj zarzad GitHub w kwestiach strategicznych, w szczegolnosci w zakresie budowy oferty enterprise

### Marc Andreessen

**Marc Andreessen**, wspolzalozyciel funduszu, byl publicznie wspierajacy transakcje i wielokrotnie wypowiadal sie pozytywnie o modelu biznesowym GitHub oraz jego roli w ekosystemie technologicznym. Andreessen postrzegal GitHub jako kluczowy element infrastruktury internetowej nowej generacji.

## 5. Dzialania po inwestycji (model "high-touch")

Andreessen Horowitz jest znany z tzw. modelu operacyjnego "high-touch", polegajacego na aktywnym wspieraniu spolek portfelowych nie tylko kapitalem, ale rowniez zasobami operacyjnymi, doradczymi i sieciowymi. W przypadku GitHub wsparcie to obejmowalo nastepujace obszary:

### Rekrutacja kadry zarzadczej

a16z aktywnie pomagal GitHub w rekrutacji kluczowych menedzerow, w szczegolnosci liderow ds. sprzedazy korporacyjnej (enterprise sales leadership). Zespol rekrutacyjny a16z wykorzystywal swoja rozlegla siec kontaktow w Dolinie Krzemowej do identyfikacji i pozyskania kandydatow na stanowiska kierownicze, ktorzy posiadali doswiadczenie w skalowaniu organizacji sprzedazowych w firmach SaaS.

### Strategia wejscia na rynek korporacyjny (Enterprise Go-to-Market)

Zespoly operacyjne a16z wspieraly GitHub w opracowaniu i realizacji strategii wejscia na rynek korporacyjny (enterprise go-to-market strategy). Obejmowalo to:

- Definiowanie segmentow docelowych klientow korporacyjnych
- Projektowanie modeli cenowych dla produktu GitHub Enterprise
- Budowe procesow sprzedazy bezposredniej i kanalow partnerskich
- Opracowanie materialow marketingowych skierowanych do decydentow IT w duzych organizacjach

### Skalowanie zespolow inzynierskich

Doradztwo w zakresie skalowania organizacji inzynierskiej stanowilo istotny element wsparcia. a16z pomagal w kwestiach takich jak:

- Struktura organizacyjna zespolow inzynierskich
- Procesy rekrutacji i onboardingu inzynierow na skale
- Najlepsze praktyki w zakresie zarzadzania produktem i priorytetyzacji prac rozwojowych
- Budowa kultury inzynierskiej sprzyjajacaj innowacyjnosci i jakosci kodu

### Budowa marki i wsparcie PR

a16z wykorzystal swoja siec medialna i komunikacyjna do wspierania budowy marki GitHub w szerszym srodowisku biznesowym. Fundusz pomagal w:

- Pozycjonowaniu GitHub jako platformy nie tylko dla indywidualnych deweloperow, ale rowniez dla duzych korporacji
- Organizacji wystapien medialnych i konferencyjnych dla zarzadu GitHub
- Budowie narracji wokol GitHub jako kluczowego elementu transformacji cyfrowej przedsiebiorstw

### Wprowadzenia strategiczne

a16z wykorzystywal swoje relacje z duzymi klientami korporacyjnymi do organizowania strategicznych wprowadzen (strategic introductions), umozliwiajacych GitHub bezposredni dostep do decydentow w firmach z listy Fortune 500. Te kontakty przyspieszaly cykl sprzedazy i pomagaly w budowie referencji korporacyjnych.

## 6. Follow-on (inwestycje kontynuacyjne)

### Historia rund finansowania GitHub

- **Zalozenie (2008):** GitHub rozpoczal dzialalnosc bez zewnetrznego finansowania VC, korzystajac z wlasnych srodkow zalozycieli (bootstrapping)
- **Series A (lipiec 2012):** 100 milionow dolarow od Andreessen Horowitz -- jedyna wieksa runda venture capital w historii spolki

### Brak dodatkowych rund VC

GitHub nie przeprowadzil dodatkowych tradycyjnych rund finansowania venture capital po inwestycji a16z w 2012 roku. Firma byla w duzej mierze samowystarczalna dzieki przychodom generowanym z:

- Platnych kont indywidualnych i zespolowych
- Licencji GitHub Enterprise (on-premises)
- Subskrypcji chmurowych dla organizacji

Model biznesowy GitHub generowal wystarczajace przychody, aby finansowac dalszy rozwoj bez koniecznosci pozyskiwania dodatkowego kapitalu zewnetrznego. Bylo to nietypowe dla spolek technologicznych o tak wysokiej wycenie w tamtym okresie.

### Utrzymanie pozycji przez a16z

Andreessen Horowitz utrzymywal swoja pozycje inwestycyjna w GitHub przez caly okres od rundy Series A w 2012 roku az do przejecia przez Microsoft w 2018 roku. Fundusz nie dokonywal dodatkowych inwestycji follow-on (ze wzgledu na brak kolejnych rund), ale nie dokonywal rowniez czescowej sprzedazy udzialow na rynku wtornym (secondary market), utrzymujac pelna ekspozycje na spolke az do momentu wyjscia (exit).

## Podsumowanie

Inwestycja Andreessen Horowitz w GitHub Inc. stanowi modelowy przyklad udanej inwestycji venture capital w sektorze narzedzi deweloperskich i infrastruktury oprogramowania. Kluczowe wnioski z tej transakcji obejmuja:

1. **Timing inwestycji** -- a16z zainwestowal w momencie, gdy GitHub osiagnal juz product-market fit i generowal przychody, ale potrzebowal wsparcia strategicznego do ekspansji na rynek korporacyjny.

2. **Wartosc dodana poza kapitalem** -- model operacyjny a16z, obejmujacy wsparcie w rekrutacji, strategii GTM i budowie marki, przyczynily sie do przyspieszenia rozwoju GitHub Enterprise.

3. **Cierpliwosc inwestycyjna** -- utrzymanie pozycji przez 6 lat bez presji na przedwczesne wyjscie pozwolilo na pelna realizacje potencjalu inwestycji.

4. **Skala zwrotu** -- dziesieciokrotny zwrot z inwestycji (10x) w ciagu 6 lat, przekladajacy sie na wartosc okolo 1 miliarda dolarow, plasuje te transakcje wsrod najbardziej udanych w historii funduszu.

Przejecie GitHub przez Microsoft za 7,5 miliarda dolarow w 2018 roku potwierdzilo teze, ze platformy stanowiace infrastrukture dla spolecznosci deweloperskiej moga osiagac wyceny na poziomie najwiekszych spolek technologicznych. Inwestycja ta pozostaje jednym z najczesciej cytowanych przykladow w analizach strategii inwestycyjnej Andreessen Horowitz.
